# User Stories – Swipr

### 1) Student Registration 
As a student,  
I want to create an account,  
so that I can access the platform and view job vacancies.

**Acceptance Criteria**
- The registration form allows entering first name, last name, email, and password
- The student can successfully register with valid input
- A confirmation message is shown after successful registration


### 2) Student Login 
As a student,  
I want to log in to my account,  
so that I can access my dashboard and job matches.

**Acceptance Criteria**
- The login form accepts email and password
- Valid credentials grant access to the student dashboard
- An error message is shown if credentials are invalid


### 3) View Job Vacancies (Student)  
As a student,  
I want to view available job vacancies,  
so that I can decide which jobs interest me.

**Acceptance Criteria**
- A list of available vacancies is displayed
- Each vacancy shows relevant job information
- Vacancies are only visible to logged-in students


### 4) Like / Match with a Vacancy
As a student,  
I want to like or skip job vacancies,  
so that I can express my interest in specific jobs.

**Acceptance Criteria**
- The student can like or skip a vacancy
- Liked vacancies are stored as potential matches
- Skipped vacancies are not shown again


### 5) View Matched Vacancies (Student) 
As a student,  
I want to view my matched vacancies,  
so that I can see which companies showed interest in me.

**Acceptance Criteria**
- A list of matched vacancies is available
- Only confirmed matches are shown
- The student can access match details


---

### 6) Company Registration
As a company,  
I want to create an account,  
so that I can post job vacancies and find students.

**Acceptance Criteria**
- The registration form allows entering company details and contact information
- The company can successfully register with valid data
- A confirmation message is shown after registration


### 7) Company Login
As a company,  
I want to log in to my account,  
so that I can manage my profile and vacancies.

**Acceptance Criteria**
- The login form accepts email and password
- Valid credentials grant access to the company dashboard
- An error message is shown if login fails


### 8) Create Job Vacancy
As a company,  
I want to create a job vacancy,  
so that students can view and match with it.

**Acceptance Criteria**
- A vacancy form allows entering job details
- The vacancy is saved and published successfully
- The vacancy becomes visible to students


### 9) View Own Vacancies (Company)
As a company,  
I want to view my posted vacancies,  
so that I can manage them easily.

**Acceptance Criteria**
- The company sees a list of its own vacancies
- Each vacancy displays its status
- Only the company’s own vacancies are shown


### 10) Delete Job Vacancy
As a company,  
I want to delete a job vacancy,  
so that outdated or incorrect vacancies are removed.

**Acceptance Criteria**
- The company can delete a vacancy it created
- Deleted vacancies are no longer visible to students
- A confirmation message is shown after deletion


### 11) View Matched Students (Company) 
As a company,  
I want to view students who matched with my vacancies,  
so that I can identify potential candidates.

**Acceptance Criteria**
- A list of matched students is displayed per vacancy
- Only confirmed matches are shown
- Student details are visible to the company


---
### 12) Update User Profile
As a user,  
I want to update my profile information,  
so that my account details remain accurate.

**Acceptance Criteria**
- The user can edit their profile fields
- Changes can be saved successfully
- A confirmation message is shown after saving


### 13) Navigation Between Login and Registration
As a user,  
I want to easily navigate between login and registration pages,  
so that I can quickly access the correct page.

**Acceptance Criteria**
- Login pages contain a link to registration
- Registration pages contain a link to login
- Navigation works correctly
